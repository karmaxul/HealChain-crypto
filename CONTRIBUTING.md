# Contributing to HealChain

Thank you for your interest in contributing to HealChain. We welcome contributions across the cryptographic research, SDK, documentation, and smart contract layers.

---

## What We Welcome

- **Bug reports** — especially in the SDK, API, or smart contract behavior
- **SDK improvements** — JavaScript (`@healchain/sdk`) and Python (`healchain-sdk`)
- **Documentation** — corrections, clarifications, examples
- **Research contributions** — related to ci-sha4096, ci-poseidon, or Reed-Solomon implementations
- **Smart contract audits** — HealChainStorage v6 on any supported network
- **Vertical page improvements** — UI, accessibility, mobile experience

## What We Are Not Currently Accepting

- Changes to core oracle infrastructure (closed source)
- Changes to billing or payment processing logic
- New chain deployments (handled internally by the team)

---

## How to Contribute

1. **Open an issue first** before submitting a pull request for anything non-trivial. Describe what you want to change and why.
2. **Fork the repository** and create a branch from `main`.
3. **Make your changes** with clear, focused commits.
4. **Test your changes** — for SDK changes, ensure existing tests pass.
5. **Submit a pull request** with a clear description of what changed and why.

We review all pull requests but cannot guarantee merge timelines. Please be patient.

---

## Code Style

- **Go:** follow standard `gofmt` formatting
- **JavaScript/TypeScript:** follow existing ESLint config in the SDK
- **Solidity:** follow existing NatSpec documentation style
- **Markdown:** clear, concise, no unnecessary formatting

---

## Reporting Security Vulnerabilities

**Do not open a public issue for security vulnerabilities.**

Report security issues privately to [security@healchain.org](mailto:security@healchain.org). See [SECURITY.md](SECURITY.md) for full details.

---

## Commit Messages

Write commit messages in the imperative: "Add feature X", not "Added feature X" or "Adds feature X". Keep the first line under 72 characters.
